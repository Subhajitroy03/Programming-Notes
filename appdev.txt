DESIGN A APP: 
When the button is clicked, it checks the content of the editText:
If the editText is NOT empty: It takes the user's input, makes the myTextView visible, and displays the user's input in the myTextView.
If the editText IS empty: It makes the myTextView visible and displays a generic pre-defined message (retrieved from resources) in the myTextView.

private lateinit var myTextView: TextView
private lateinit var editText: EditText

override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    setContentView(R.layout.activity_main)

    myButton = findViewById(R.id.myButton)
    myTextView = findViewById(R.id.myTextView)
    editText = findViewById(R.id.editText)

    myButton.text = getString(R.string.button_text)

    myButton.setOnClickListener {
        if (!editText.text.toString().isEmpty()) {
            val input = editText.text.toString()

            myTextView.visibility = View.VISIBLE
            myTextView.text = input
        } else {
            myTextView.visibility = View.VISIBLE
            myTextView.text = getString(R.string.text_view_text_message)
        }
    }
}

package com.example.seekbar

import android.os.Bundle
import android.widget.SeekBar
import android.widget.TextView
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity

// Assuming a class structure for an Android Activity
class MainActivity : AppCompatActivity() {

    // Properties for the UI elements
    private lateinit var seekBar: SeekBar
    private lateinit var textViewProgress: TextView // Corrected variable name as seen in the code logic

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // Initializing the UI elements
        // Assuming the SeekBar ID is R.id.seekBar based on usage
        seekBar = findViewById(R.id.seekBar)
        textViewProgress = findViewById(R.id.textViewProgress)

        // Setting the listener for the SeekBar
        seekBar.setOnSeekBarChangeListener(object : SeekBar.OnSeekBarChangeListener {

            // Called when the progress level has changed
            override fun onProgressChanged(p0: SeekBar?, p1: Int, p2: Boolean) {
                // Show a toast with the event name
                Toast.makeText(this@MainActivity, "onProgressChanged", Toast.LENGTH_SHORT).show()
                // Update the TextView with the current progress rate
                textViewProgress.text = "Rate = " + p0!!.progress.toString() // p0 is the SeekBar instance
            }

            // Called when the user starts touching the SeekBar
            override fun onStartTrackingTouch(p0: SeekBar?) {
                // Show a toast with the event name
                Toast.makeText(this@MainActivity, "onStartTrackingTouch", Toast.LENGTH_SHORT).show()
            }

            // Called when the user stops touching the SeekBar
            override fun onStopTrackingTouch(p0: SeekBar?) {
                // Show a toast with the event name
                Toast.makeText(this@MainActivity, "onStopTrackingTouch", Toast.LENGTH_SHORT).show()
                // Update the TextView with the final progress rate after releasing
                textViewProgress.text = "Final Rate = " + p0!!.progress.toString() // p0 is the SeekBar instance
            }
        })
    }
}

ACTIVITY LIFE CYCLES 

Here are the main Android Activity Lifecycle callback methods and their meaning in simple English, presented point-wise:

onCreate(): The screen is created. This is when the app sets up the screen layout and initializes variables.

onStart(): The screen is becoming visible. The user can now see the screen, but it's not yet active for input.

onResume(): The screen is fully active and ready to use. This is the main state where the user interacts with the app.

onPause(): The screen is losing focus. Something else (like a pop-up or a phone call) is partially covering it. You should save any important data quickly.

onStop(): The screen is completely hidden. The user has switched to another app or screen, and the system is saving resources.

onRestart(): The hidden screen is waking up. The user is navigating back to this screen after it was stopped.

onDestroy(): The screen is being destroyed. This is the final cleanup before the screen is removed from memory.

eg -
package com.example.lifecycledemo

import android.os.Bundle
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        Toast.makeText(this@MainActivity, "onCreate called", Toast.LENGTH_SHORT).show()
    }

    override fun onStart() {
        super.onStart()
        Toast.makeText(this@MainActivity, "onStart called", Toast.LENGTH_SHORT).show()
    }

    override fun onResume() {
        super.onResume()
        Toast.makeText(this@MainActivity, "onResume called", Toast.LENGTH_SHORT).show()
    }

    override fun onPause() {
        super.onPause()
        Toast.makeText(this@MainActivity, "onPause called", Toast.LENGTH_SHORT).show()
    }

    override fun onStop() {
        super.onStop()
        Toast.makeText(this@MainActivity, "onStop called", Toast.LENGTH_SHORT).show()
    }

    override fun onRestart() {
        super.onRestart()
        Toast.makeText(this@MainActivity, "onRestart called", Toast.LENGTH_SHORT).show()
    }

    override fun onDestroy() {
        super.onDestroy()
        Toast.makeText(this@MainActivity, "onDestroy called", Toast.LENGTH_SHORT).show()
    }
}


You need to override Activity Lifecycle functions for three main reasons:

Resource Management: To save battery and improve performance by starting (e.g., music playback in onStart()) and stopping (e.g., pausing music in onStop()) tasks when the screen becomes visible or hidden.

State Preservation: To ensure the user doesn't lose data by saving unsaved input in onPause() and restoring it in onCreate().

User Experience (UX): To handle interruptions smoothly, making sure the app pauses and resumes correctly when interrupted by something like a phone call.


SWITCHING FROM ONE SCREEN TO ANOTHER

FIRST CREATE ANOTHER Empty Activity file and change as per you need 
Now, on first page upon clicking a button it should go to second page 

Here is the code for it

package com.example.activities

import ...

class MainActivity : AppCompatActivity() {
    private lateinit var goToButton: Button

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        goToButton = findViewById(R.id.button_go_to_act)

        goToButton.setOnClickListener { view!
            val intent = Intent(packageContext: this@MainActivity, SecondActivity::class.java)
            startActivity(intent)
        }
    }
}



1. val intent = Intent(packageContext: this@MainActivity, SecondActivity::class.java)
This line creates an Intent, which acts like a request or a message used to communicate between Android components. Think of it as a ticket with a destination written on it.

val intent: Declares a new, immutable variable named intent.

Intent(...): This calls the constructor to create the Intent object.

packageContext: this@MainActivity: This specifies where the request is coming from. this@MainActivity refers to the current screen (the MainActivity).

SecondActivity::class.java: This specifies where the request is going. It tells the Android system to load and start the class named SecondActivity.

2. startActivity(intent)
This line is the instruction that tells the Android system to carry out the request defined by the intent.

startActivity(...): This is a method inherited from the Activity class.

(intent): The argument passed is the "ticket" you just created, which specifies the destination (SecondActivity).

When this line executes, the Android system reads the intent, finds the SecondActivity, and launches it, bringing it to the foreground and hiding the MainActivity (moving it into the onPause() and onStop() states).

HOW TO PASS DATA FROM ONE ACTIVITY TO ANOTHER

First store the keys in a separate kotlin file:

package com.example.activities

class Constants {
    companion object {
        const val INTENT_MESSAGE_KEY = "Message"
        const val INTENT_MESSAGE2_KEY = "Message2"
        const val INTENT_DATA_NUMBER = "Number"
    }
}

inside MainActivity.kt

package com.example.activities

import ...

class MainActivity : AppCompatActivity() {
    private lateinit var goToButton: Button

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        goToButton = findViewById(R.id.button_go_to_act)

        goToButton.setOnClickListener { view!
            val intent = Intent(packageContext: this@MainActivity, SecondActivity::class.java)
            intent.putExtra(Constants.INTENT_MESSAGE_KEY, value: "Hello from first Activity")
            intent.putExtra(Constants.INTENT_MESSAGE2_KEY, value: "How was your day?")
            intent.putExtra(Constants.INTENT_DATA_NUMBER, value: 3.14)
            startActivity(intent)
        }
    }
}

inside SecondActivity.kt


package com.example.activities

import ...

class SecondActivity : AppCompatActivity() {
    private lateinit var textViewDataIntent: TextView

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_second)

        textViewDataIntent = findViewById(R.id.textViewData)

        val data = intent.extras   /// extracting data from second page

        data?.let {
            val message = data.getString(Constants.INTENT_MESSAGE_KEY)
            val message2 = data.getString(Constants.INTENT_MESSAGE2_KEY) //giving key to extract the value
            val number = data.getString(Constants.INTENT_DATA_NUMBER)

            textViewDataIntent.text = message + "\n" + message2 + "\n" + number
        }
    }
}

putExtra is a method used to attach additional data (called extras) to the Intent.

key: A string identifier for the data. (Must be unique to avoid overwriting)

value: The data you want to send (can be String, Int, Double, Boolean, Parcelable, etc.)

class MainActivity : AppCompatActivity() {
    private lateinit var goToButton: Button
    private lateinit var textViewResult: TextView

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        val getResult =
            registerForActivityResult(ActivityResultContracts.StartActivityForResult()) {
                if (it.resultCode == Constants.RESULT_CODE) {
                    val message = it.data!!.getStringExtra(Constants.INTENT_MESSAGE2_KEY)
                    textViewResult.text = message
                }
            }

        textViewResult = findViewById(R.id.textView)
        goToButton = findViewById(R.id.button_go_to_act)

        goToButton.setOnClickListener {
            val intent = Intent(this@MainActivity, SecondActivity::class.java)
            intent.putExtra(Constants.INTENT_MESSAGE_KEY, "Hello from first Activity")
            intent.putExtra(Constants.INTENT_MESSAGE2_KEY, "How was your day?")
            intent.putExtra(Constants.INTENT_DATA_NUMBER, 3.14)
            getResult.launch(intent)
        }
    }
}
